# MVPリリースまでに必要なタスク

## 目的

メグループ のMVPでは、ユーザーがGoogleログイン後に自分の定期タスクを登録し、次回実施日を確認し、完了記録をカレンダーで振り返れる状態をリリース基準とする。

## MVPスコープ

### 含める機能

- Google OAuth によるログイン、ログアウト、認証コールバック
- 定期タスクの一覧表示
- 定期タスクの新規登録、編集、削除
- タスクの完了操作と `maintenance_logs` への履歴保存
- 完了履歴のカレンダー表示
- マイページでのユーザー情報表示
- 利用規約、プライバシーポリシーへの導線

### MVPでは含めない機能

- プッシュ通知、メール通知、リマインダー配信
- タスクの共有、家族・チーム利用
- 画像アップロード、添付ファイル管理
- 高度な検索、タグ、フィルタ、並び替え
- 課金、プラン管理

## リリースブロッカー

| 優先度 | タスク | 完了条件 |
| --- | --- | --- |
| P0 | 本番Supabaseプロジェクトを準備する | Auth、DB、RLS、Google OAuth設定が本番環境で動作する |
| P0 | 本番環境変数を整備する | VercelにSupabase URL、Anon Key、Google OAuth関連値が設定されている |
| P0 | Googleログインの本番リダイレクトURLを設定する | 本番URLからログインして `/auth/callback` 経由でアプリに戻れる |
| P0 | タスク削除UIを実装する | ユーザーが不要なタスクを削除でき、RLSで本人データのみ削除される |
| P0 | 利用規約ページを作成する | ログイン画面のリンク先が `#` ではなく `/terms` になる |
| P0 | プライバシーポリシーページを作成する | ログイン画面のリンク先が `#` ではなく `/privacy` になる |
| P0 | 主要導線の手動確認を完了する | 新規ログイン、作成、編集、完了、履歴確認、削除が通る |
| P0 | `yarn lint`、`yarn typecheck`、`yarn test:run`、`yarn build` を通す | CI相当の品質確認がローカルまたはCIで成功する |

## 機能実装タスク

| 優先度 | 領域 | タスク | 完了条件 |
| --- | --- | --- | --- |
| P0 | 認証 | 未ログイン時の保護ルート挙動を確認する | `/dashboard`、`/calendar`、`/mypage`、`/create_task`、`/task/[id]` が未ログイン時に安全に扱われる |
| P0 | 認証 | ログアウト導線を確認または実装する | ユーザーが明示的にログアウトでき、セッション終了後にログイン画面へ戻る |
| P0 | タスク管理 | タスク削除ボタンと確認UIを追加する | 誤操作を防ぎつつ削除でき、削除後は一覧とカレンダーが更新される |
| P0 | タスク管理 | タスク完了操作のフィードバックを整える | 完了後にカード表示が更新され、成功・失敗がユーザーに伝わる |
| P1 | タスク管理 | 作成・編集フォームの日本語文言を確認する | 入力ラベル、バリデーション、エラー文言が自然な日本語で表示される |
| P1 | カレンダー | 完了履歴とタスク名の紐付け表示を確認する | 履歴一覧で対象タスクが分かり、削除済みタスクでも表示が破綻しない |
| P1 | マイページ | 通知設定UIの扱いを決める | MVP対象外なら非表示、表示するなら未実装に見えない状態にする |
| P1 | 空状態 | 初回利用時の案内を整える | タスク未登録時に作成導線が分かりやすく表示される |
| P2 | アクセシビリティ | キーボード操作とフォーカス表示を確認する | 主要操作をキーボードで完了できる |
| P2 | レスポンシブ | モバイル幅で主要画面を確認する | ログイン、一覧、フォーム、カレンダーでレイアウト崩れがない |

## データベース・セキュリティタスク

| 優先度 | タスク | 完了条件 |
| --- | --- | --- |
| P0 | `maintenance_items` の本番マイグレーションを適用する | 本番DBにテーブル、RLS、`updated_at` トリガーが存在する |
| P0 | `maintenance_logs` の本番マイグレーションを適用する | 本番DBにテーブル、RLS、検索用インデックスが存在する |
| P0 | RLSの動作をユーザーA/Bで確認する | 他ユーザーのタスクとログを参照・更新・削除できない |
| P0 | Service Role Keyの露出がないことを確認する | クライアントバンドルや公開環境変数に秘密鍵が含まれない |
| P1 | 入力値の上限を確認する | タスク名、メモ、アイコン、周期に過大入力を入れても破綻しない |
| P1 | エラーログ方針を整理する | 本番でユーザー向け文言と開発者向けログの役割が分かれている |

## 推奨実施順

1. README、仕様書、環境変数ドキュメントの文字化けを直し、開発・確認手順を読める状態にする。
2. 本番SupabaseとGoogle OAuthを準備し、本番相当のログインを成立させる。
3. タスク削除、ログアウト、規約・プライバシー導線などのリリースブロッカーを実装する。
4. タスク作成、編集、完了、カレンダー履歴、削除の一連の流れを手動確認する。
5. lint、typecheck、test、buildを通し、失敗があればMVP範囲の不具合だけ修正する。
6. Vercelへデプロイし、本番URLでリリース前チェックリストを再実施する。
