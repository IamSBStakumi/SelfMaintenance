# SelfMaintenance - MVPリリーススケジュール (8週間プラン)

本ドキュメントは、SelfMaintenanceアプリケーションの機能を約2ヶ月（8週間）でMVPとしてリリースするための週毎のタスクリストです。

## 目標

ユーザーがアカウントを作成し、自分自身のメンテナンスタスク（例：コンタクトレンズ交換、通院など）を登録・閲覧・完了できるコア機能を一貫して提供できる状態を目指します。

---

### Week 1: 開発基盤構築と認証機能

基本となるプロジェクト構造と、必須となるユーザー認証システムを確立します。

- [x] 開発環境の確認（Next.js App Router, Tailwind CSS v4, Vitest, lint-staged等）
- [x] Supabase と Next.js プロジェクトの接続・環境変数設定
- [x] Supabase Auth を利用したサインアップ / ログイン処理の実装
- [x] スプラッシュ画面 および ログイン/新規登録画面 のUI構築

### Week 2: データベース設計とAPI・DB周辺の実装

バックエンドにおけるデータの保存設計と読み書きの口を用意します。

- [x] Supabase の `maintenance_items` テーブル等の構築およびマイグレーション作成
- [x] Row Level Security (RLS) ポリシーの設定・検証（各ユーザーがセキュアにデータを扱えるかのテスト）
- [x] 更新日時等に関わるデータベーストリガーの作成
- [x] 各種テーブルに対するCRUD関数（サーバーアクションまたはバックエンド連携処理）の実装

### Week 3: ホーム画面およびタスク機能の実装

アプリケーションの要となるダッシュボードと、タスクの登録・表示機能を作ります。

- [x] ホーム画面（メインダッシュボードレイアウト）のUI実装
- [x] 各タスク（`maintenance_items`）の一覧表示機能の実装
- [x] タスク追加機能の実装
- [x] タスク詳細閲覧画面の実装
- [x] ボタン押下によるタスクの「完了時記録（`last_completed_at` の更新）」のロジック実装

### Week 4: マイページ・設定機能の実装

個人に紐づいた継続的な設定データの管理機能を実装します。

- [ ] ユーザー設定や特定用品のデータ（コンタクトレンズ・眼鏡の度数など）を保持する設計とテーブル追加
- [ ] レンズの種類や度数を保存・編集できるUIの作成
- [ ] 通知設定などのアカウント付随の項目のUIの実装
- [x] マイページの結合およびホーム画面からの遷移パス整備

### Week 5: カレンダーおよびスケジュール機能の基礎構築

「いつ何をやるか／いつ何をやったか」を視覚的に把握する機能を作成します。

- [x] アプリ内カレンダーUIコンポーネントの実装
- [x] タスクの完了履歴をカレンダー上に表示・マッピングする処理の作成
- [ ] 通知スケジュールのロジック検討方針決定（バッチや定時チェックの仕組み、通知許可等の権限）

### Week 6: 通院ログと処方箋メモの実装

健康管理・メンテナンスの履歴情報を保存する仕組みを追加します。

- [ ] カレンダー画面等と連携した「通院ログ」「処方箋メモ」のUI表示
- [ ] 画像や書類データを含むログ入力機能の実装（必要に応じて Supabase Storage の連動）
- [ ] 通院結果等の保存とカレンダー上への表示確認

### Week 7: 全体連携テストとUI/UXのブラッシュアップ

出来上がった各機能をつなぎ合わせ、ユーザービリティ改善とバグの洗い出しを実施します。

- [ ] 各画面間の遷移、状態保持などの機能結合テストの実施
- [ ] エラーハンドリング・ローディング時のフィードバック等、体験面のUI改善
- [ ] スマートフォンでの表示崩れを中心としたレスポンシブ対応の調整
- [ ] 主要ロジックに対するユニットテスト（Vitest）の拡充とカバレッジ向上

### Week 8: 本番環境へのデプロイメントとリリース

外部に向けてプロダクトを安定供給し、運用に乗せるための最終段階です。

- [x] Vercel などのホスティング環境へのデプロイ構築と本番用環境変数の設定
- [x] 本番 Supabase 環境における RLS や外部公開範囲などの最終セキュリティ確認
- [ ] アプリケーション全体のエンドツーエンドな動作確認
- [ ] 最初のテスターに向けた MVPリリース
